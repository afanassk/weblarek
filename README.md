# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

В приложении используются следующие интерфейсы данных:

### `IProduct` — интерфейс товара

Описывает данные товара, представленного в каталоге интернет-магазина. Содержит основную информацию, необходимую для отображения товара на странице и оформления заказа: название, описание, категорию, изображение и стоимость.

```typescript
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}
```

### `IBuyer` — интерфейс покупателя

Описывает данные покупателя, необходимые для оформления заказа. Содержит контактную информацию и способ оплаты.

```typescript
type TPayment = 'card' | 'cash';

interface IBuyer {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}
```

### `IOrder` — интерфейс заказа

Описывает данные заказа, необходимые для его оформления и отправки на сервер. Содержит сведения о покупателе, выбранных товарах и общей стоимости покупки.

```typescript
interface IOrder {
  payment: TPayment;       
  email: string;
  phone: string;
  address: string;
  total: number;
  items: string[];         // массив id товаров
}
```

## Модели данных

Ниже приведены описания назначения, конструкторов, полей и методов с указанием типов:

### `ProductsModel` — модель каталога товаров

Отвечает за хранение и управление данными каталога. Сохраняет полный список доступных товаров и информацию о товаре, выбранном для детального просмотра.

Поля класса:
- `productsList: IProduct[]` — массив всех товаров каталога.
- `selectedProduct: IProduct | null` — товар, выбранный для подробного отображения.

Методы класса:
- `setProducts(products: IProduct[]): void` — сохраняет массив товаров в модель.
- `getProducts(): IProduct[]` — возвращает массив всех товаров.
- `getProductById(id: string): IProduct | undefined ` — возвращает товар по идентификатору.
- `setSelectedProduct(id: string): void` — сохраняет товар для детального просмотра (карточка/модалка).
- `getSelectedProduct(): IProduct | null` — возвращает выбранный для просмотра товар.

### `CartModel` — модель корзины покупок

Отвечает за управление товарами, добавленными пользователем в корзину. Обеспечивает добавление и удаление товаров, а также расчёт их общего количества и итоговой стоимости.

Поля класса:
`products: IProduct[]` — массив товаров, добавленных в корзину.

Методы класса:
- `getCartProducts(): IProduct[]` — возвращает массив товаров в корзине.
- `addProductToCart(product: IProduct): void` — добавляет товар в корзину (не добавляет товар, если его цена равна null — такие товары считаются недоступными к покупке).
- `removeProductFromCart(product: IProduct): void ` — удаляет из корзины все вхождения товара с данным id.
- `clearCart(): void` — полностью очищает корзину.
- `getTotalCartPrice(): number` — возвращает общую стоимость всех товаров в корзине.
- `getTotalCartProducts(): number` — возвращает количество товаров в корзине.
- `isProductInCart(id: string): boolean` — проверяет наличие товара по идентификатору.

### `BuyerModel` — модель данных покупателя

Отвечает за хранение и обработку информации, введённой покупателем при оформлении заказа. Выполняет валидацию полей и проверку корректности введённых данных.

Поля класса:
`buyerData: IBuyer | null` — частично заполненные данные покупателя (позволяет обновлять отдельные поля).

Методы класса:
- `setBuyerData(detail: Partial<IBuyer>): void` — сохраняет данные покупателя (можно передавать как все поля, так и любое подмножество полей; уже сохранённые значения не стираются).
- `getBuyerData(): IBuyer | null` — возвращает текущие сохранённые данные покупателя.
- `clearBuyerData(): void` — очищает все поля данных покупателя.
- `sumAddressErrors(): {[key: string]: string}` - проверяет, заполнен ли адрес доставки.
- `sumContactsErrors(): {[key: string]: string}` - проверяет email и телефон покупателя.

## Слой коммуникации

### `AppApi` - класс для работы с API сервера

Отвечает за обмен данными между приложением и сервером через HTTP-запросы. Реализует композицию с базовым классом Api, используя его методы для выполнения запросов к серверу.

Конструктор:

```typescript
constructor(api: IApi) {
  this.api = api;
}
```

Поля класса:  
`private api: IApi` — экземпляр базового класса Api, используемый для выполнения HTTP-запросов к серверу.

Методы класса: 
- `getProductList(): Promise<IProduct[]>` - выполняет GET-запрос для получения списка товаров с сервера.
- `createOrder(order: IOrder): Promise<IOrderResponse>` - выполняет POST-запрос для отправки данных заказа на сервер.

## Слой представления (View)

### `Card` - карточка товара

Назначение: это родительский класс для карточек.

Поля класса:
- `id?: string` — идентификатор товара;
- `titleElement` - элемент для отображения названия товара;
- `priceElement` - элемент для отображения цены товара;
- data - данные товара для рендеринга дочерних классов.

Методы класса:
- `set title(value: string)` - устанавливает название товара;
- `set price(value: number | null)` - (поддерживает значение «Бесценно», если цена null).

### `CardBasket` - элемент товара в корзине

Назначение: отвечает за отображение данных товара в корзине.

Поля класса: 
- `productIndex` - порядковый номер товара в корзине;
- `deleteButtonElement` - кнопка удаления товара из корзины.

Методы класса: 
- `set index(value: number)` - устанавливает порядковый номер товара в корзине;
- render - рендерит карточку по данным в data

### `CardCatalog` - карточка товара в каталоге

Назначение: отвечает за отображение карточки в каталоге.

Поля класса:  
- `categoryElement` - элемент для отображения категории товара;
- `imageElement` - элемент для отображения изображения товара.

Методы класса: 
- `set category(value: string)` - устанавливает текст категории и CSS-классы согласно categoryMap;
- `set image(value: string)` - устанавливает изображение товара;
- render - рендерит карточку по данным в data.

### `CardPreview` - детали карточки товара

Назначение: отвечает за превью карточки после нажатия на карточку в каталоге.

Поля класса: 
- `imageElementt` - изображение товара;
- `categoryElement` - категория товара;
- `descriptionElement` - описание товара;
- `buttonElement` - кнопка добавления/удаления товара из корзины;
- `inBasket` - булево значение, находится ли товар в корзине.

Методы класса: 
- `set image(value: string)` - устанавливает изображение товара;
- `set category(value: string)` - устанавливает текст категории и CSS-классы согласно categoryMap;
- `set description(value:string)` - устанавливает текст описания товара;
- render - рендерит карточку по данным в data.

### `HeaderView` - отображение шапки приложения

Назначение: управляет отображением счетчика товаров в корзине и обработкой клика по корзине.

Поля класса:
- `counterElement` - элемент для отображения количества товаров в корзине;
- `basketButton` - кнопка корзины для открытия модального окна.

### `GalleryView` - отображение каталога товаров

Назначение: отвечает за отображение каталога, состоящего из карточек товаров.

Поле класса: `catalogElement` - контейнер, в который рендерятся карточки товаров.

Метод класса: `set catalog(items: HTMLElement[])` - принимает массив карточек и заменяет содержимое каталога на переданные элементы.

### `ModalView` - управление модальными окнами

Назначение: контролирует отображение и скрытие модальных окон, обработку закрытия.

Поля класса:
- `isOpen: boolean = false` - 
- `closeButtonElement` - кнопка закрытия модального окна;
- `contentElement` - контейнер для содержимого модального окна;
- `modalContainer` - корневой элемент модального окна (оверлей).

Методы класса:
- `open()` - открывает модальное окно, добавляя CSS-класс modal_active;
- `close()` - закрывает модальное окно, удаляя CSS-класс и очищая содержимое;
- `set content(value: HTMLElement)` - устанавливает содержимое в модальное окно;

### `BasketView` - отображение корзины покупок

Назначение: Показывает список товаров в корзине, общую стоимость и кнопку оформления.

Поля класса:
- `titleElement` - заголовок модального окна корзины;
- `listElement` - контейнер для списка товаров корзины;
- `priceElement` - элемент для отображения общей стоимости;
- `buttonElement` - кнопка перехода к оформлению заказа.

Методы класса:
- `set buttonText(value: string)` - меняет текст кнопки оформления товара в корзине;
- `set basket(items: HTMLElement[])` - устанавливает список товаров в корзине;
- `set total(value: number)` - устанавливает общую стоимость товаров в корзине;
- `set buttonDisabled(value: boolean)` - включает или отключает кнопку оформления заказа, например, если корзина пуста.

### `Form` - форма 

Назначение: это родительский класс для форм.

Поля класса: 
- `errorElement` - элемент для отображения ошибок формы;
- `submitButtonElement` - кнопка отправки формы.

Методы класса: 
- `set submitButtonDisabled` - включает или отключает кнопку отправки формы;

### `FormOrder` - форма заказа

Назначение: отвечает за отображение формы заказа.

Поля класса:

Методы класса:
- `selectPayment` - выбор способа оплаты

- `set address(value: string)` - устанавливает значение адреса доставки.

### `FormContacts` - форма контактов

Назначение: отвечает за отображение формы контактов.

Поля класса:
- `emailInputElement` - поле ввода email;

Методы класса:
- `set email(value: string)` - устанавливает значение email;

### `SuccessView`- окно успешного оформления заказа

Поля класса: 
- `buttonElement` - кнопка закрытия окна успеха;
- `descriptionElement` - элемент для отображения суммы списания;

Методы класса: 
- `set total(value: number)` - устанавливает сумму списания в текст сообщения.

## Презентер (обработка событий)

В приложении обрабатываются следующие события:

- `catalog:changed` - вызывается при изменении модели каталога; презентер заново создаёт карточки товаров и рендерит их в галерее.
- `card:select` - срабатывает при клике по карточке в каталоге; презентер находит выбранный товар в модели каталога и открывает модальное окно с превью товара.
- `card:add-product` - добавляет выбранный товар в корзину, обновляет модель корзины и счётчик товаров в шапке.
- `card:remove-product` - удаляет товар из корзины, обновляет модель корзины и счётчик в шапке.
- `cart-counter:changed` - обновляет счётчик товаров в корзине в шапке (компонент `Header`).
- `cart:open` - открывает корзину: презентер формирует список карточек товаров корзины, считает общую сумму и передаёт все данные в компонент `Basket` для отображения в модальном окне.
- `cart:order` - инициирует первый шаг оформления заказа: открывается форма выбора способа оплаты и ввода адреса (`FormOrder`).
- `cart:contacts` - переключает приложение на второй шаг оформления: открывается форма ввода контактных данных (`FormContacts`).
- `cart:success` - обрабатывает успешное завершение заказа: получает от сервера сумму списания, очищает корзину и данные покупателя, открывает окно подтверждения (`SuccessView`) с итоговой суммой.
- `modal:close` - закрывает модальное окно и, при необходимости, сбрасывает временное состояние интерфейса.
